# Support

This project is supported through the repository issue tracker and maintainer
review. Support is intentionally lightweight: the goal is to keep the project
usable, not to promise a staffed help desk.

## Where To Ask

- Bug reports: use the bug issue template in `.github/ISSUE_TEMPLATE/bug_report.yml`
- Documentation requests: use the documentation issue template
- Feature requests: use the feature request issue template
- Pull request feedback: use the normal review flow

## For Maintainers

- The maintainer contact path is the one described in `CODEOWNERS` and the
  Code of Conduct.
- Use private contact for conduct issues, security-sensitive reports, or other
  matters that should not be handled in a public issue.

## What To Include

- Package name and version
- Operating system and runtime version
- Minimal reproduction steps
- Error messages or logs
- Whether the issue affects Python, Rust, R, Julia, Go, C#, or TypeScript

## Expectations

- Issues are handled best effort.
- Public support channels are for reproducible bugs, feature requests, and
  documentation gaps.
- External community review packets should be tracked in the community
  readiness docs rather than in support requests.
